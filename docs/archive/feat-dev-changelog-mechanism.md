# feat: 建立多 Agent 协作变更记录机制

- 变更时间：2026-06-10 14:45 (+08:00)

## 变更理由
用户诉求：仓库需支持多 Agent 协作。希望每次提交代码时，在变更记录目录下
生成一个 Markdown 文件记录变更（时间 / 理由 / 内容），并让后续 Agent 能
通过浏览该目录了解项目历史。

## 变更内容
- AGENTS.md（新增）：在根目录定义变更记录机制——命名两段式
  `<类型>-<描述>.md`、内容三要素（时间/理由/内容）、模板，以及"查看历史"
  的说明，指引 Agent 开工前先浏览变更记录目录。
- .gitignore：为变更记录目录添加例外（`tmp/` 仍忽略，但该子目录纳入版本
  控制），确保记录随仓库提交、多 Agent clone 后可见。
- 变更记录目录：补建上述机制建立前已完成的 3 个提交的记录，作为初始历史。
